# AGENTS.md

Working-principles and repo-map conventions for AI agents working in this repo. See `CLAUDE.md` for the data architecture, page↔script map, and PWA/offline behavior — this file complements it rather than repeating it.

## Working principles

- **Smallest change that solves the request.** Don't refactor, restyle, or "clean up" code beyond what was asked, even if you notice unrelated issues nearby — mention them instead of fixing them inline.
- **Don't broaden scope.** If the task is to fix one page/feature, don't touch unrelated modules. Each feature page owns its own script and CSS; cross-file "consistency" edits are out of scope unless explicitly requested.
- **Match the existing file's style.** There is no shared linter or formatter — conventions (naming, quote style, comment density) are per-file. Mirror the file you're editing for anything purely stylistic.
- **Prefer a shared helper over duplicating logic.** If the same fetch/parse/business logic is needed in more than one file, extract it into its own small `<script>`-included file (see CLAUDE.md's "Shared conventions" section and the `db-adapter.js`/`home-layout.js`/`kwiedza-data.js` pattern) instead of copy-pasting it. The only exception is when sharing would force a page to load or offline-precache code/data it doesn't need, purely for the sake of reuse — don't do that.
- **No build step, ever.** Don't introduce a bundler, package manager, transpiler, or dependency that requires a build. Every page must keep working as plain `<script>` tags served statically (see CLAUDE.md's "Working in this repo" section for how to actually run/test it).

## Repo map (beyond the page↔script table in CLAUDE.md)

| Path | Purpose |
|---|---|
| `assets/` | Bundled static content shipped in this repo: kWiedza docs (`assets/docs/index.json` + markdown), app icons, splash screens, OG images |
| `icns_transit/` | SVG icons for transport line types (IC/metro/regio/etc.), used by `map.js` and `route-search.js` |
| `icns_ui/` | Generic UI icons (search, settings, edit, delete, ...) shared across feature pages |
| `mc-items/` | PNG renders of Minecraft items, used by kHandel product cards |
| `map/base`, `map/tiles2x`, `map/tiles4x` | Pre-rendered map tile layers (base + 2x/4x hi-res, light/dark) served directly by `map.js`; generated by `temporary_scripts/generate_map_assets.py`, not hand-authored |
| `ksejm/data`, `ksejm/docs` | Bundled index + real markdown/PDF acts for kSejm — part of this repo, not the external `kobywatel-db` (see CLAUDE.md's data architecture section) |
| `retro/` | Standalone legacy/Easter-egg page, not part of the main navigation or the page↔script table — keep it isolated, don't wire it into shared JS |
| `temporary_scripts/` | One-off local Python tooling (e.g. map asset generation); not deployed, never referenced from HTML/JS |
| `.claude/` | Local Claude Code settings for this repo |
